# CAN_Reverse_Engineering_Project- Part B
# Craig Atkinson - a1669436

## Research Objective
Develop a Transformer Model that can automatically classify each bit of a CAN Message (64 bits) with a functional data type (counter/checksum/sensor/flag bit).

## Secondary Objectives

### 1. Architecture Optimization & Comparison
- Evaluate multiple transformer architectures using the already decoded message IDs :
  - Test Temporal input sizes (sliding windows of 50, 100, 200, 500 consecutive messages)
  - Test different CNN layers  for local (2d) feature extraction
  - Test Different attention mechanisms (self-attention, cross-attention variants)
  - Test positional encoding effect
- Determine the optimal transformer configuration for bit-level CAN message classification

### 2. Vehicle-Specific Model Development
- Manually decode CAN message IDs for Subaru and Toyota vehicles as initial test cases
- Create labeled datasets from decoded CAN messages (using CANdid message DB from 3 more vehicles)
- Use open-source OBD reference materials and data exploration to manually decode messages
- Train and validate models separately for each vehicle model to establish baseline performance

### 3. Generalization Testing
- Test model generalisation capabilities:
  - Across different vehicles from the same OEM (same OEM generalisation)
  - Across vehicles from different OEMs (different OEM generalisation)
- Test models on unseen vehicles/OEMs

### 4. Combined vs. Separate Vehicle Datasets Model Comparison
- Develop and test models that uses datasets from multiple vehicles, and multiple OEM datasets
- Compare performance of:
  - Models trained separately on individual vehicles/OEMs
  - Models trained on combined datasets from multiple vehicles/OEMs
- Do combined models outperform vehicle-specific approaches?

### Timeline (2026 Term 1)
- **(Week 3-4)**: Manual Decoding, Transformer Architecture experimentation
- **(Week 5-6)**: Generalization testing, combined model approaches
- **(Week 7-10)**: Comparison of Combined Datasets, further optimisation, documentation, final reporting
